BTB3D 深读：3D 医学 VLM 的瓶颈，真的是语言主干不够大吗？

导读。要把 3D 医学 VLM 做强，关键究竟在“换更大的语言主干”，还是在“如何把一卷三四百张切片的 CT 压成一串既紧凑又保真、还能 2D/3D 通用的 token”？BTB3D 的回答是后者。它用两条机制独立的下游验证同一件事：看图写报告和按文本生成 CT 都变好时，瓶颈更像在视觉 token，而不是 LLM。

一句话贡献小波频域压缩 + 因果 3D 卷积 + lookup-free 量化 + 三阶段课程，把 3D 医学 VLM 的核心变量从“语言主干大小”改写为“体素 token 的保真与可扩展”。
关键机制CT 体先做 3D Haar 小波，再用严格因果的 3D encoder-decoder 产生 LFQ 离散 token；短体训练后用重叠时间 tiling 扩展到长 CT，Stage 3 冻结编码器只精修解码器。
核心证据报告生成 Ours-16 F1 0.258 vs CT-CHAT 0.184（+40% 相对）；text-to-CT mean FID 2.236 vs GenerateCT 9.512（约 −76.5%）。
复现门槛重建披露 64×H100，报告生成 40×H100，text-to-CT 16×H100；总 GPU 小时/墙钟时间：论文未说明。

问：为什么开篇不先讲报告生成分数，而先问 token？

引导：如果只看一个下游，很容易把提升归因到 prompt、LoRA 或解码头；如果理解侧和生成侧共用同一 tokenizer 同时提升，归因才更集中。

答：BTB3D 的核心论证不是“又一个报告生成模型更强”，而是“3D 医学 VLM 的视觉 token 更保真后，两类不同任务都会受益”。所以本页先把 tokenization 当作核心问题，再看表格与图像证据。

Takeaway：这篇论文的读法是先追问“信息怎么进模型”，再看 LLM 或生成 head 的表现。

1. 动机：3D 医学 VLM 为什么卡在视觉 token？

3D CT 比 2D X 光包含更多轴向上下文，天然适合自动写放射报告、按文本合成体数据、做教学或规划。但现有 3D VLM 落后于 2D 同类，BTB3D 把病根放在视觉侧：对比学习式编码器、slice-wise tokenization、以及生成侧 encoder-decoder 都没有同时解决“长序列、细解剖、2D/3D 通用、可重建”这几件事。

第一处错位是 CLIP 式对比学习。放射报告不是“每张图只有唯一正确文本”的自然图像 caption：不同医生可用不同措辞描述同一病征。把“未配对但相关”的报告当负样本惩罚，会放大标注噪声。并且高分辨率 3D 体极吃显存，CLIP 式大 batch 与深模型很难同时保住，最后容易得到欠拟合的弱编码器。

信息瓶颈视角。对比目标优化的是“图-文互信息”的判别代理，只保留能区分配对/非配对的粗粒度语义，不保证保留下游报告里的细粒度发现。重构目标则要求隐变量 $z$ 能还原 $x$，迫使 token 保留细解剖。苏剑林在变分编码/信息瓶颈中点明“重构损失希望隐变量尽量保留信息” [archives/6181]，正对应这里为什么要从对比换成重构。

第二处错位是 slice-wise tokenization。2D 医学 VLM 不能直接吃体数据，只能把 3D 投影成 2D 切片，于是丢掉轴向上下文；3D 数百切片又带来长序列建模难题；配对 3D 数据稀缺，也让从头训练大 3D 模型更难。

第三处错位在生成侧。GenerateCT 式级联方法会出现层间不连续，MedSyn 式轻量 encoder-decoder 保真度不够。根因仍然是缺一个能在长高分辨率序列上高保真重建的 encoder-decoder。

问：如果语言主干足够大，能不能自动补回视觉 token 丢掉的细节？

引导：报告里的小结节、细微纹理、层间连续性先存在于 CT 体里；如果压缩时已经丢掉，语言模型只能靠先验猜。

答：BTB3D 的判断是不能。真正缺的是一种 encoder-decoder：不丢细节地扩展到长体序列，用统一表示弥合 2D/3D，并能从该表示解码出高分辨率 3D 体。理解侧和生成侧都会受益，这也呼应苏剑林在多模态设计里强调的“以最保真的方式喂入、再用重构学表示”的思路 [archives/9984]。

Takeaway：把算力投在更保真、可 2D/3D 统一、能流式扩展的 token 上，比单纯堆更大 LLM 更划算。

2. 贡献：BTB3D 明确提出了什么？

作者明确提出的贡献集中在视觉 tokenization：一个因果卷积 encoder-decoder，为每卷 CT 学一串紧凑的频率感知体素 token；一个三阶段训练课程，使模型能“短切片上训、>300 切片上用”，且不增额外显存；一个统一 2D/3D 的因果设计，让单切片可视为长度 1 的体。

下游结果则服务于同一个中心论断：精确的三维 tokenization，而不是更大的语言主干，才是可扩展 3D 医学 VLM 的关键。报告生成端，临床 F1 较 CT-CHAT 提升 40%；text-to-CT 端，FID 相比 GenerateCT/MedSyn 大幅下降，能生成解剖一致的 512×512×241 体。

问：这篇的新意是某个单点模块，还是模块组合？

引导：小波、因果卷积、LFQ、课程学习都不是孤立炫技；它们分别压住显存、2D/3D 通用、码本效率和长序列泛化。

答：新意更像一个面向 3D 医学体的工程组合：用可逆频域变换省显存，用因果结构统一 2D/3D，用 LFQ 避免大码本查表，用三阶段课程解决短训长用。任何一环都服务于“token 保真且可扩展”。

Takeaway：BTB3D 的贡献不是把 LLM 换大，而是把 3D 视觉 token 的生成、压缩、量化、长序列推理串成闭环。

3. 数据：CT-RATE 训练，RadChestCT 域外验证

CT-RATE 是最大的公开 3D 医学配对报告数据集：25,692 卷胸部 CT / 21,304 名患者；训练 20,000 患者，测试 1,304。体数据转 Hounsfield Units 并裁剪到 [−1000, 1000]。Stage 1&2 使用原始体；Stage 3 重采样到 0.75×0.75×1.5 mm，并裁/填到 512×512×241。

RadChestCT 用于外部验证，提供多标签二值标签，但没有文本报告，所以只报告 F1、Precision、Recall。报告生成使用 findings + impression 段；所有临床指标用官方 CT-RATE 报告分类器计算。

Takeaway：CT-RATE 负责配对报告与生成主任务，RadChestCT 负责域外二值标签检验；两个数据集回答的问题不同。

4. Pipeline：CT 体如何变成可重建 token？

一句话流程是：CT 体 → 3D Haar 小波频域压缩 → 因果 3D 卷积编码器 → LFQ 离散 token → 因果解码器 → 逆小波 → 重建体。下游把 token 接到 LLaMA（报告生成）或 transformer + 交叉注意力（text-to-CT）。

  1. 输入。CT 体先进入 HU 裁剪与尺度处理，再进入 3D Haar 小波域。
  2. 局部子体重建。Stage 1 让 2D 单切片或 9 切片子体端到端学习局部时空结构。
  3. 重叠时间 tiling。Stage 2 用 9 切片窗、8 步滑动产生长序列 token，减少整卷训练显存压力。
  4. 长序列解码器精修。Stage 3 冻结编码器和码本，只微调解码器，增强长程解剖一致性。

问：为什么 Stage 2 不直接把每个窗口所有 token 都拼起来？

引导：9 切片窗之间高度重叠，全部保留会引入冗余，也会让长序列更难处理。

答：BTB3D 只保留每个窗口的第二个 token，因为它已经聚合该窗 9 片信息；首窗保留两个 token 用于起始覆盖。这样既保持长程表示一致，又避免把重复窗口边界带入潜序列。

Takeaway：重叠 tiling 的关键不是“切小块”，而是只保留已聚合上下文的代表 token，从而短训长用。

5. 方法：小波、因果卷积、LFQ 与三阶段课程

5.1 3D Haar 小波频域压缩

对 $x\in\mathbb{R}^{D\times H\times W}$ 做 3D Haar 小波 $\mathcal{W}(\cdot)$：每个 $2\times2\times2$ 体素块变成 8 个子带（1 个低频 + 7 个高频），每轴分辨率减半，通道 $1\to8$，但体积缩 $8\times$。低频保粗结构，高频保细解剖边界，所以它是“频率感知”的压缩。2D 切片（$D=1$）下同一变换退化为低/高频对，保证 2D/3D 架构一致。小波正交可逆，压缩只是换基，意图是把“省显存”和“保细节”解耦。低秩/压缩“少存元素保主信息”的思路，可类比苏剑林 SVD 自编码器一文 [archives/4208]；这里是类比，归因较弱。

5.2 因果 3D 卷积与因式分解

编码器 $E$ 用残差块，卷积分解为 $1\times k\times k$ 空间卷积和 $k\times1\times1$ 时间卷积。轴向使用因果 padding：过去补 $k-1$ 个零，未来不补，因此第 $t$ 个 token 只看 $\le t$ 的切片，第一个 token 仅由第一张切片算出。这让“单切片（2D）= 长度 1 的体（3D）”成立，也有利于下游自回归。

压缩配置有两种：8×8×8 配置用两层 stride-2，使空间/时间各 $4\times$，叠加小波 $2\times$ 得到 $8\times$ 有效压缩；16×16×8 配置多一层空间 stride-2，使空间 $16\times$、时间 $8\times$。前者保更多空间细节，后者在显存紧或全局语义任务中更合适。因果性是否给非自回归重建带来不必要损失：论文未做因果 vs 双向卷积对照，论文未说明。

5.3 Lookup-free quantization 与熵正则

编码器输出 $y=E(\mathcal{W}(x))$ 经 LFQ 离散化：码数 $K=2^d$，每个特征维按符号二值化 $b\in\{-1,1\}^d$，打包成整数 token $z\in\{0,\dots,K-1\}$。它不需要码本/embedding 查表，所以训练和推理更省显存。论文取 $d=18$，码本大小 $2^{18}=262{,}144$。熵正则 $\mathcal{L}_{\text{entropy}}$ 鼓励 $K$ 个码均匀使用，防止码本坍缩。

5.4 重构式训练目标

解码器 $G$ 镜像编码器，重建到小波域 $\tilde{\mathcal{W}}(x)$，再经逆小波 $\mathcal{W}^{-1}$ 得到 $\hat{x}$。总损失为：

$$\mathcal{L}=\mathcal{L}_{\text{rec}}+\lambda_{\text{adv}}\mathcal{L}_{\text{adv}}+\mathcal{L}_{\text{vq}}$$
式 1：BTB3D 的重构式训练目标。
  • $\mathcal{L}_{\text{rec}}=\mathbb{E}_x\|x-\hat{x}\|_1$，用 $\ell_1$ 保持更锐利的细解剖。
  • $\mathcal{L}_{\text{adv}}=\mathbb{E}_x[-\log D(\hat{x})]$，3D 判别器 $D$ 的监督直接作用在 CT 域，而不是小波域。
  • $\mathcal{L}_{\text{vq}}=\mathbb{E}\big[\|\mathrm{sg}[y]-e\|_2^2+\beta\|y-\mathrm{sg}[e]\|_2^2\big]$，$\mathrm{sg}$ 为停梯度。
  • 论文显式省略 VGG 感知损失，因为自然 RGB 与灰度医学图不匹配，可能反而损害医学重建。

这一目标与“重构损失希望隐变量尽量保留信息” [archives/6181]、对抗 + 互信息正则做编码生成 [archives/6214] 在思路上相承。

5.5 三阶段课程

Stage 1 在单 2D 切片或 9 切片子体上端到端训练 $(E,Q,G)$，学习局部时空结构；Stage 2 使用 9 切片窗和 8 步滑动，$T=\lfloor(D-1)/8\rfloor$，最终潜序列为 $[z^1_1,z^1_2,z^2_2,z^3_2,\dots,z^T_2]$；Stage 3 冻结 $E$ 与码本，只微调解码器 $G$，增强长程解剖一致性。推理可 one-shot 或 tiled，下游均使用 tiled。

问：这套方法里最能连接“理解”和“生成”的部件是哪一个？

引导：理解任务需要 token 保留临床发现，生成任务需要 token 能被解码成连续 CT 体；两者共享的是同一 encoder-decoder 表示。

答：核心是可重构的频率感知 token。报告生成把 token 映入 LLaMA 输入空间，text-to-CT 则用 transformer + 交叉注意力在同一潜空间生成，再由解码器还原 CT。若 token 只保粗语义，两个下游不会同时受益。

Takeaway：BTB3D 的方法链条服务于同一个目标：让视觉 token 既紧凑又能还原细解剖。

7. 主实验：同一 tokenizer 能否同时撑住理解与生成？

论文有三问：（1）三阶段课程是否优于朴素端到端？（2）该 tokenization 能否提升报告生成等 VLM 任务？（3）BTB3D 在从临床 prompt 合成高分辨率 CT 上较 SOTA 如何？本节先看两条机制独立、共享同一 tokenizer 的验证路径。

路径 1：理解侧。固定 LLaMA-3.1-8B 与 CT-CHAT 配置，只换 token，因此报告生成差异更集中地归因到 tokenizer。路径 2：生成侧。同一 encoder-decoder 表示接 transformer + 交叉注意力做 text-to-CT。两端都赢，能排除“只是某个下游 head 调得好”的替代解释。

问：为什么 Ours-16 在报告生成最好，而 Ours-8 在生成 CT 最好？

引导：两个任务对 token 的偏好不同：报告生成偏全局语义，像素级合成偏空间细节。

答：16×16×8 压得更狠，token 更偏全局语义，适合报告生成；8×8×8 保留更多空间细节，适合 text-to-CT 的像素级合成。这种交叉结果反而支持“压缩率—任务”的权衡假设。

Takeaway：同一 tokenizer 家族在理解和生成两端都提升，但最佳压缩率随任务目标改变。

8. 消融：三阶段课程到底带来什么？

三阶段课程解决的是“短训长用”的矛盾：只在短序列上训，整卷后段会退化；直接分块推理，窗口边界又容易不连续。Table 1 显示 Stage 2 的重叠 tiling 带来最大跳变，Stage 3 解码器精修继续小幅稳定提升。

这个可证伪预测在两类主结果里也能相互印证：16×16×8 在报告生成最好，8×8×8 在生成最好。这说明压缩率不是越低越好或越高越好，而要跟下游任务对语义/空间细节的需求匹配。

问：Table 1 里最关键的跳变为什么在 Stage 2，而不是 Stage 3？

引导：Stage 3 只精修解码器；Stage 2 改变的是长序列训练/推理方式。

答：Stage 2 引入重叠 tiling，让模型第一次在长 CT 的潜序列上学习全卷一致性，所以 PSNR、SSIM、MSE 都出现大幅跳变。Stage 3 冻结编码器和码本，只让解码器适配长程结构，因此更像稳定锐化。

Takeaway：三阶段课程的核心收益来自 Stage 2 的长序列表示一致性，Stage 3 是在此基础上的解码器修补。

问：这篇最容易被过度宣传成什么？

引导：注意它验证的是胸部 CT、相对增益和 token 瓶颈，不是临床部署完成。

答：容易被说成“3D 医学 VLM 已经可用”或“一个 tokenizer 通吃所有 3D 医学影像”。更严谨的说法是：BTB3D 在胸部 CT 上证明，更保真、可扩展的视觉 token 能同时提升报告生成和 text-to-CT，但临床绝对 F1、跨解剖泛化、因果卷积对照和成本披露仍有限。

Takeaway：论文强在定位瓶颈和给出两条独立验证，不强在宣告临床系统已可直接落地。

结语：几点学术判断

一句话总结：在数据稀缺、序列超长、显存吃紧的 3D 医学 VLM 场景，把算力投在更精确、更保真、可 2D/3D 统一、能流式扩展的 token 上，比堆更大语言主干更划算。BTB3D 用“小波频域压缩 + 因果卷积 + LFQ + 三阶段课程”的工程组合，把“保真 token”哲学 [archives/9984] 在两条机制独立的下游（报告生成 F1 / text-to-CT FID-FVD）上同时兑现。

三个 takeaway：
  1. 视觉 token 是主变量。固定 LLaMA-3.1-8B 与 CT-CHAT 配置后，只换 token 仍带来临床 F1 提升。
  2. 重构目标补上对比学习的短板。对比学习保粗粒度配对语义，重构目标强迫 token 保留可解码的细解剖。
  3. 压缩率要跟任务走。16×16×8 更适合报告生成，8×8×8 更适合 text-to-CT；这不是矛盾，而是任务偏好的证据。

问：如果后续要反驳 BTB3D，最该补哪两个实验？

引导：找论文未说明或未验证的关键替代解释。

答：第一是因果卷积 vs 双向卷积的等算力对照，确认因果性不是重建负担；第二是跨解剖/跨模态验证，确认胸部 CT 上的 token 结论能否迁移到脑 MRI、腹部 CT 等场景。

Takeaway：BTB3D 的中心论断成立在当前证据内；最重要的外推边界是架构对照和数据域扩展。
