DeltaTok: how one delta token per frame makes generative world modeling cheap enough to sample

Paper2Html2026-06-10CVPR 2026 Top Paper (candidate) · Amazon / Eindhoven University of Technology / Johns Hopkins UniversitySource: arXiv:2604.04913 PDF
World ModelBest-of-ManyDelta TokenVFM Feature SpaceDense ForecastingMathJax checked

The short version. A world model predicts what a scene may look like next. In driving and robotics, one future is not enough: a pedestrian can keep walking, stop, or turn. A deterministic model usually collapses that uncertainty into one average prediction. A diffusion-style generative model can sample many futures, but it pays for that diversity with repeated forward passes over dense spatial tokens. DeltaTok changes the representation. It encodes the feature difference between two consecutive frames into one continuous delta token. DeltaWorld then predicts those delta tokens with a Best-of-Many objective: generate many hypotheses in parallel, but supervise only the closest one. Because a training frame is satisfied the moment any single hypothesis lands on the real future, the other hypotheses are free to specialize to different outcomes instead of all regressing to the average. The sampled delta tokens are then decoded back to VFM features. At 512 x 512 resolution, one delta token replaces a 32 x 32 grid of patch tokens, a 1024x per-frame token reduction. On dense forecasting, DeltaWorld's best-of-20 sample beats Cosmos-4B and Cosmos-12B on every reported metric while using 3.1 x 104 GFLOPs instead of 6.0-6.4 x 107 — about 2000x less compute. Its 20-sample mean is competitive but not uniformly ahead; the win is in the best sample, which is what a multi-hypothesis forecaster is for.

One-line contributionDeltaTok compresses the change between consecutive DINOv3 feature maps into one token, and DeltaWorld uses those tokens to sample multiple plausible futures in one forward pass.
Key mechanismBest-of-Many removes the "average future" pressure, while delta compression removes the dense spatial token grid that makes multi-hypothesis training expensive.
Core evidenceOn the 512 x 512 dense forecasting table, DeltaWorld uses about 2000x fewer GFLOPs than Cosmos while scoring higher in best-of-20 VSPW, Cityscapes, and KITTI metrics.
Reproduction boundaryThe method operates in frozen VFM feature space, not directly in pixels. It relies on DINOv3 features, the Best-of-Many query budget, and sequential DeltaTok decoding.

1. The core problem: why is a generative world model so expensive?

A world model is useful only if its futures are plausible enough for downstream decisions. A single regression prediction is fragile when the scene has multiple valid next states. It tends to land near the center of the possible futures, which can be a feature-space average that no real future follows.

Generative models fix the single-future problem by sampling. The cost then moves elsewhere. If each future is a full grid of spatial tokens, every sample requires predicting a large feature map. If the sampler is iterative, one future also requires many forward passes. The paper targets both costs at once: use a single-pass multi-hypothesis objective, and make each future small enough that many hypotheses are affordable.

Q: What is the smallest object a future predictor can reasonably predict?

Guide: do not start from pixels. The paper predicts in the feature space of a vision foundation model, or VFM, where each frame is already represented as semantic patch tokens.

A: A full frame token is still too much for one token, because it must carry the whole scene. A frame difference is smaller. Consecutive frames usually share the same road, buildings, sky, and layout. The useful information is how the scene changes. DeltaTok makes that the modeling unit: one token says how to transform the previous VFM feature map into the next one.

Takeaway: The paper's compression claim depends on temporal redundancy. It is strongest when the next frame is mostly the previous frame plus structured change.

2. What the authors actually claim

The stated contributions have two parts. First, the authors propose DeltaTok, a tokenizer that encodes only the change between consecutive frame features as a single delta token. With a DINOv3 ViT-B feature map at 512 x 512 input resolution, this turns 1024 patch tokens per frame into one token. Second, they introduce DeltaWorld, a generative world model that predicts future delta tokens with Best-of-Many training. During training, many futures are generated in parallel and only the closest one is supervised. At inference, the same mechanism yields diverse futures in one forward pass.

The larger inference is a cost decomposition. "Generative world model" is not inherently expensive. It becomes expensive when diversity requires repeated sampling and each sample is a dense spatial map. DeltaWorld removes the repeated sampling cost with Best-of-Many and removes most of the spatial cost with delta tokens. The price is that the model now forecasts in VFM feature space, then relies on downstream heads or decoders to turn those features into task outputs or images.

3. Data and tasks: forecasting in frozen VFM feature space

The paper evaluates dense forecasting, not open-ended video generation. A four-frame context is given. The model predicts a short-term future directly, about 0.2 seconds ahead, and a mid-term future by a three-step autoregressive rollout, about 0.6 seconds ahead. Segmentation is measured by mIoU, where higher is better. Depth is measured by RMSE, where lower is better. The predicted object is a DINOv3 feature map; fixed task heads convert that feature map into segmentation or depth predictions.

4. Pipeline: from frames to delta tokens to sampled futures

Q: Why not compress every frame into one absolute token and predict that?

A: The paper tries that as an intermediate step. It is efficient, but it asks one token to represent the whole scene at every time step. That loses capacity exactly where futures differ. Delta compression changes the job. The token only needs to encode the update from \(x_{t-1}\) to \(x_t\). That is why the final model improves over frame compression in both best and mean mIoU in Table 2.

Takeaway: Single-token compression works only after the token is assigned the right job: encode the temporal change, not the entire spatial state.

5. The method: Best-of-Many plus delta compression

The baseline is DINO-world, a deterministic predictor in VFM feature space. A frozen VFM \(\phi\) maps each video frame \(v_i\) to a grid of patch tokens \(x_i=\phi(v_i)\in\mathbb{R}^{H\times W\times D}\). The predictor forecasts each future patch token independently, conditioned on all context features and timestamps.

$$\hat{x}_{t+1,h,w}=f\!\left(q,\,X_{1:t},\,T_{1:t},\,\tau_{t+1},\,h,\,w\right)\in\mathbb{R}^{D}.$$
Eq. 1: DINO-world-style deterministic prediction for one spatial location.

How to read Eq. 1

\(q\) is a learnable query, \(X_{1:t}\) is the context sequence of VFM feature maps, \(T_{1:t}\) are their timestamps, \(\tau_{t+1}\) is the target time, and \((h,w)\) is the patch position. The expensive part is hidden in the index: this prediction is repeated for every \(H\times W\) patch location.

Best-of-Many turns the deterministic predictor into a one-pass sampler. Instead of one query, it samples \(K\) noise queries from a Gaussian. Each query proposes a different future.

$$q^{k}\sim\mathcal{N}(\mu,\Sigma),\qquad k=1,\dots,K.$$
Eq. 2: stochastic queries for multi-hypothesis prediction.
$$\hat{x}^{k}_{t+1,h,w}=f\!\left(q^{k},\,X_{1:t},\,T_{1:t},\,\tau_{t+1},\,h,\,w\right)\in\mathbb{R}^{D}.$$
Eq. 3: the \(k\)-th query produces the \(k\)-th candidate future patch token.

The loss then chooses the candidate closest to the ground truth. Only that candidate receives supervision.

$$\begin{split} k^\star &= \arg\min_{k}\sum_{h,w}\ell\!\left(x_{t+1,h,w},\,\hat{x}^{k}_{t+1,h,w}\right);\\ L_{\text{BoM}} &= \sum_{h,w}\ell\!\left(x_{t+1,h,w},\,\hat{x}^{k^\star}_{t+1,h,w}\right). \end{split}$$
Eq. 4: Best-of-Many objective with smooth L1 loss \(\ell\).

Why "best" avoids the average future

A regression model is rewarded for being close to all possible futures, so it can land near a central prediction. Best-of-Many changes the contract. A training example is satisfied if one query lands near the realized future. Different queries can therefore specialize to different modes. The assumption is that many queries get chances to win over training. If the same small subset wins forever, the unused queries receive no gradient and diversity can collapse.

BoM still becomes expensive if each query predicts a whole spatial map. The paper first introduces a frame-level tokenizer that compresses an entire feature map into one frame token.

$$z_t=g(x_t,z_{\mathrm{init}})\in\mathbb{R}^{D}.$$
Eq. 5: frame-level encoder compressing one feature map into one token.
$$\hat{x}_t=h(x^{\mathrm{init}},z_t).$$
Eq. 6: frame-level decoder reconstructing a feature map from the frame token.
$$L_{\mathrm{tok}}=\lVert x_t-\hat{x}_t\rVert^2.$$
Eq. 7: tokenizer reconstruction loss.

The frame-token variant is the key negative control. It makes BoM cheaper, but it forces one token to store a whole frame. DeltaTok changes that capacity problem by conditioning on the previous frame.

$$z_t=g(x_{t-1},x_t,z_{\mathrm{init}})\in\mathbb{R}^{D}.$$
Eq. 8: DeltaTok encoder, where \(z_t\) encodes the change from \(x_{t-1}\) to \(x_t\).
$$\hat{x}_t=h(x_{t-1},z_t).$$
Eq. 9: DeltaTok decoder, which reconstructs the current feature map using the previous feature map and the delta token.

Finally, DeltaWorld predicts future delta tokens directly. A black frame is prepended so that the first token \(z_1\) encodes the absolute first real frame. After that, the predictor sees a sequence \(Z_{1:t}=(z_1,\dots,z_t)\) and predicts the next delta token.

$$\hat{z}_{t+1}=f(q^{k},Z_{1:t},T_{1:t},\tau_{t+1}).$$
Eq. 10: DeltaWorld future predictor in one-dimensional delta-token space.

Q: What is the two-angle argument that delta tokens should help?

A: From the representation angle, adjacent frames have high redundancy, so the difference is lower-dimensional than the whole frame. From the training-cost angle, BoM needs many candidates; making each candidate one token lets the loss be computed in token space without decoding every spatial map during predictor training. The ablation table tests both angles: frame compression buys efficiency but hurts capacity, while delta compression keeps the efficiency and recovers accuracy.

Takeaway: DeltaTok is not just a smaller latent. It is a smaller target whose definition matches the temporal structure of video.

6. Training and compute ledger

The implementation is deliberately tied to a frozen feature backbone. All experiments use DINOv3 ViT-B features. DeltaTok itself is a continuous autoencoder, not a variational autoencoder. The tokenizer and predictor also use ViT-B configurations, but the paper states that the formulation is not restricted to this scale.

8. Ablations: which design actually carries the result?

The cleanest ablation builds the system in four steps. Step 0 is deterministic DINO-world. Step 1 adds Best-of-Many but still predicts full spatial features. Step 2 compresses each full frame to one token. Step 3 replaces full-frame compression with delta compression.

Q: do the ablations prove that delta compression is the mechanism, rather than just smaller compute?

A: They make a strong case. Frame compression is also small, but it hurts Cityscapes best and mean scores. Delta compression is similarly compact, yet it improves over frame compression by 6.0 mIoU best and 10.0 mIoU mean on Cityscapes in Table 2. The discriminative checks then show that delta tokens can preserve accuracy while cutting time, memory, and token count outside the BoM model.

Takeaway: The useful compression is not "one token" by itself. It is one token for the right residual signal.

9. Limitations and reproduction cost: where to be careful

Table E. Boundaries stated or implied by the paper.
BoundaryWhat the paper says or showsHow to read it
Distribution modelingBoM lacks the explicit distributional objective that diffusion denoising provides.The model can sample distinct futures, but the sample probabilities are not guaranteed to match the true predictive distribution.
Query coverageCoverage is limited by the number of noise queries \(K\) explored during training, and there is no explicit mechanism forcing diverse use of query space.Figure 6 supports useful diversity in the tested range, but a different dataset or larger future space may need a different \(K\) or an explicit diversity objective.
Error accumulationAbsolute feature maps are reconstructed by repeatedly decoding delta tokens conditioned on previous features, so errors may compound across steps.Longer rollouts are the natural stress test. The paper suggests training the tokenizer on its own reconstructions as a mitigation, but this is future work.
Feature-space forecastThe model predicts DINOv3 features and evaluates them through fixed downstream heads.The evidence is strong for dense forecasting in VFM space. It is not the same claim as high-fidelity pixel video generation.
Temporal redundancy assumptionDeltaTok assumes consecutive frames differ in structured, usually low-dimensional ways.Hard cuts, violent camera motion, or large scene transitions weaken the compression prior. The paper notes that low-redundancy cases can force the token toward absolute compression.
Training data and baseline reproducibilityDINO-world code and training data are unavailable, so the paper uses a reimplementation and a roughly 4M-sample video training set.The comparison is carefully described, but exact reproduction depends on rebuilding that training corpus and protocol.

Q: What is the most likely overstatement of this paper?

A: "One token can generate a future video." The precise claim is narrower and more useful: in frozen VFM feature space, one delta token can represent the change between consecutive frames well enough to make multi-hypothesis dense forecasting cheap.

Takeaway: DeltaWorld is a feature-level generative forecaster. Its strength is efficient diversity under temporal redundancy; its boundary is distributional calibration and long-rollout drift.

10. Closing: the useful mental model

In one sentence: DeltaTok makes generative world modeling cheap by predicting residual feature changes, and Best-of-Many turns that compact residual sequence into multiple plausible futures in one pass.

Three takeaways:
  1. Deterministic dense forecasting fails when the future is multi-modal, because one output must stand in for several possible futures.
  2. Best-of-Many supplies diversity without iterative diffusion, but it is affordable only after the future representation is small.
  3. Delta tokens are small for a principled reason: adjacent VFM feature maps mostly repeat, so one token can focus on the structured change.
