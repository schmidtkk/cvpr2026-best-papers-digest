MacTok: how a few-token continuous tokenizer collapses, and how making reconstruction harder rescues it

Paper2Html2026-06-10CVPR 2026 Top Paper (candidate) · Fudan UniversitySource: arXiv:2603.29634 PDF
Image TokenizerPosterior CollapseMasked Reconstruction1D ContinuousDINO AlignmentMathJax checked

The short version. A continuous image tokenizer turns a picture into a small list of real-valued numbers, called a latent. Here we cut that list down to just 64 or 128 numbers — these are the tokens. At that size the tokenizer breaks. A VAE-style tokenizer adds a rule, the KL term, that keeps each number near one fixed default value. With so few tokens that rule wins. The encoder stops writing image content into the latent. The decoder then learns to rebuild the picture from the default alone. Those few tokens — the whole point of compressing this hard — end up empty. MacTok fixes this by making reconstruction harder. It hides image patches before encoding, so the decoder cannot guess the picture without reading the tokens. It also ties each token to features from DINOv2 — a pretrained vision model whose features are known to capture object identity and high-level structure — so the tokens carry that same high-level meaning. With 64 or 128 tokens it reaches gFID 1.44 on ImageNet 256 and 1.52 on ImageNet 512, using up to 64 times fewer tokens than the 4096-token pixel-space diffusion baselines.

One-line contributionA 1D continuous tokenizer that uses masked reconstruction plus representation alignment to stop posterior collapse at strong compression (64 or 128 tokens).
Key mechanismDINO-guided masking (DINO = DINOv2) hides the patches the model finds most informative, which forces object-level content into the latent; representation alignment then makes each token match DINOv2's features, both for the whole image and patch-by-patch; random masking adds breadth so the latent stays usable for filling arbitrary holes.
Core evidenceImageNet 256 gFID 1.44 and 512 gFID 1.52 (SiT-XL), state of the art at 64/128 tokens, up to 64x fewer tokens than the 4096-token diffusion baselines.
Reproduction costNeeds DINOv2 as both masking guide and alignment teacher; mask ratio and alignment weights need tuning; a separate decoder fine-tuning stage restores fidelity.

1. The core problem: why do few tokens collapse?

A 256×256 RGB image is 256×256×3 ≈ 200,000 pixel values — too many to generate one at a time. So generators work on a latent instead — a compressed code produced by a tokenizer. A continuous tokenizer of the VAE kind adds one rule, the KL term, that keeps each latent number close to a fixed default value. That rule buys a smooth, well-behaved latent that generators like.

The rule turns toxic once the token budget is small. Push that latent down to 64 or 128 tokens and the tokenizer breaks in a specific way. The KL term pulls each code toward the fixed default. A code at the default carries no image content, so the encoder stops writing into it. The decoder then reconstructs from the default instead. The tokens end up empty. This failure has a name: posterior collapse.

Q: KL is supposed to make the latent tidier — how does it instead squeeze the information out?

Guide: read the VAE objective as a trade-off between two jobs. One job is reconstruction, how faithfully the decoder rebuilds the image. The other is the KL term, which pushes the latent toward the default and so acts like a "forget the input" pressure.

A: It comes down to one fact: a strong decoder barely needs the latent. The KL term is smallest when the latent sits at the default, and the default carries no image information. Emptying the latent normally costs reconstruction accuracy, so the optimizer keeps some information in it. But a strong decoder can produce a plausible image on its own, leaning on what natural images usually look like. So with very few tokens and a strong decoder, using the latent or ignoring it barely changes reconstruction quality. Emptying the latent is now almost free: it sends KL to zero and costs almost no accuracy. The optimizer takes that free win, the latent falls to the default, and the encoder stops encoding.

Takeaway: Collapse is not caused by KL being too strong. It happens because reconstruction is too easy, which makes emptying the latent a free move.

2. What the authors actually claim

The paper's stated contributions are three. First, MacTok itself: a 1D continuous tokenizer that adds masked reconstruction and representation alignment to a standard KL-VAE, and so keeps the latent informative at strong compression. Second, two complementary masking strategies. Random masking hides a random subset of patches, so the latent must be usable for filling in arbitrary missing regions — this keeps it broadly informative rather than overfit to any one cue. DINO-guided semantic masking instead hides the patches DINOv2 considers most central to the object; because those patches are gone and the visible context cannot fill them, the decoder is forced to read object-level content out of the latent, so the latent has to store it. Third, global plus local representation alignment, which links the latent to DINOv2 features so the few tokens carry discriminative semantics. The combined result: 64 or 128 tokens reach gFID 1.44 on ImageNet 256 and 1.52 on ImageNet 512, with up to 64x fewer tokens than the 4096-token diffusion baselines.

The reader can infer one larger point. Earlier fixes for collapse tuned the KL weight or annealed it over training, which treats the symptom. MacTok changes the reconstruction task instead. Once reconstruction genuinely needs the latent, the free move that caused collapse no longer exists. That reframing applies to any autoencoder that compresses and then reconstructs, not only to this tokenizer.

3. Data and task: a tokenizer judged by reconstruction and by generation

The task is to train a continuous image tokenizer and then use it to support image generation. Evaluation is on ImageNet at 256×256 and 512×512. A diffusion generator — SiT-XL, with SiT-B and LightningDiT-XL also used — is trained on top of the tokenizer's latent. Two numbers carry the verdict. Reconstruction quality uses rFID, the FID between the input images and their reconstructions, plus PSNR and SSIM on 50,000 ImageNet validation images, where rFID lower is better and PSNR/SSIM higher is better. Generation quality uses gFID, the FID between generated and real images, plus Inception Score (IS), where gFID lower is better. Both gFID and IS are reported with and without classifier-free guidance (CFG), under ADM — the standard ImageNet generation-evaluation protocol. DINOv2 supplies the semantic-masking guide and the alignment target.

4. Pipeline: from a partly hidden image to a compact latent

Q: why two kinds of mask, each used half the time?

A: The two masks do different jobs. Random masking ignores content and hides a random subset of patches, so the latent must be usable for filling in arbitrary missing regions from whatever is still visible — this keeps it broadly informative. Semantic masking instead hides the patches that look most like the object, so reconstruction cannot succeed without high-level understanding, which pulls DINO's object-level knowledge into the latent. One mask buys breadth, the other buys semantic content.

Takeaway: Masking here is not data augmentation. It is a deliberate way to raise reconstruction difficulty until the latent has to do real work.

5. The method: removing the free move that causes collapse

Start from the mechanism. The continuous-tokenizer objective is reconstruction plus a KL term.

$$\mathbb{E}_{x}\Big[\underbrace{\mathbb{E}_{z\sim p(z\mid x)}[-\log q(x\mid z)]}_{\text{reconstruction / distortion}}+\underbrace{\mathrm{KL}\big(p(z\mid x)\,\|\,q(z)\big)}_{\text{KL / rate}}\Big]$$
Eq. 1: the VAE-style objective — a reconstruction term plus a KL (rate) term.

Naming the two terms (rate–distortion)

In Eq. 1 the reconstruction term is the distortion, and the KL term is the rate. This is standard rate–distortion background, not a MacTok invention. Section 1 already showed why few tokens plus easy reconstruction let the optimizer drive the rate to zero and empty the latent. The only point of naming the terms here is to anchor the five loss terms defined below to Eq. 1. With that label in place, the question becomes: how do you make distortion actually depend on the latent?

MacTok's answer is masking. Random masking replaces a fraction of image patches with mask tokens before encoding. The mask ratio $m$ is drawn from $U[-0.1,\,M]$ and clipped to $[0,M]$, with $M\approx 0.7$. The $-0.1$ lower bound puts a fixed slice of probability mass exactly at $m=0$, so a set fraction of samples land on the full, unmasked image. Those full-image samples anchor the decoder to true fidelity, so heavy masking does not wreck reconstruction. Semantic masking uses DINOv2 to score how informative each patch is, by the cosine similarity between the cls token $c$ and each patch token $p_i$, then hides the most-similar $\lfloor mN\rfloor$ patches.

$$s_i=\frac{c^{\top} p_i}{\lVert c\rVert\,\lVert p_i\rVert},\qquad \mathcal{M}_p=\mathrm{TopK}\big(\{s_i\}_{i=1}^{N},\ \lfloor mN\rfloor\big)$$
Eq. 2: DINO semantic similarity, and selecting the most informative patches to hide.

Why making reconstruction harder stops collapse

Collapse needs one precondition: the decoder can rebuild the image without the latent. Hide patches — especially the patches that carry the object — and the visible context plus the prior no longer suffice. Now the decoder must read the latent to finish the reconstruction. So distortion depends strongly on the latent, and driving KL to zero is no longer free. The masking has closed off the degenerate "latent-optional" solution. The load-bearing assumption: masking only helps if it makes reconstruction truly depend on the latent. If visible context alone can fill the gaps — the mask is too weak, or misses the important patches — the free move returns and collapse is not cured. That is exactly why the semantic mask targets the informative patches, and why the ratio must be large but not extreme.

The full training objective adds three more terms on top of reconstruction and KL. Perceptual loss compares decoder output to the input in a pretrained feature space. Adversarial loss uses a discriminator built on a frozen DINO-S backbone (a small DINO variant) to score whether the output looks like a real image; pushing the decoder to fool it keeps fine texture sharp. Representation alignment, $\mathcal{L}_{\mathrm{RA}}$, ties the latent to DINOv2 by cosine similarity — local features to the patch tokens, the global feature to the cls token — which is what injects discriminative semantics into the few tokens.

$$\mathcal{L}=\mathcal{L}_{\mathrm{recon}}+\lambda_1\,\mathcal{L}_{\mathrm{percep}}+\lambda_2\,\mathcal{L}_{\mathrm{adv}}+\lambda_3\,\mathcal{L}_{\mathrm{KL}}+\lambda_4\,\mathcal{L}_{\mathrm{RA}}$$
Eq. 3: the full objective — reconstruction + perceptual + adversarial + KL + representation alignment. Weights: $\lambda_1=1.0,\ \lambda_2=0.2,\ \lambda_3=10,\ \lambda_4=0.1$.

What each term is for, and what breaks if you drop it

The reconstruction term has weight 1 — it is the fixed anchor that sets base fidelity, and every other weight is relative to it. Drop perceptual and adversarial and the output stays correct but goes blurry — these two are what keep reconstructions sharp, with the adversarial loss using a frozen DINO-S discriminator together with DiffAug, consistency regularization, and LeCAM. The KL term ($\lambda_3=10$) is the rate pressure from Eq. 1, now safe because masking made distortion depend on the latent. The alignment term ($\lambda_4=0.1$) pours DINOv2 semantics into the latent and is what turns it from merely informative into discriminative and well-clustered.

Q: do these gains come from masking plus alignment, or is DINO distillation doing all the work?

A: Not DINO alone. Adding the modules one at a time, gFID falls 6.01 → 3.15, and dropping any single piece makes it worse. The next section shows the per-module breakdown.

Takeaway: Because no single module carries the result, you cannot drop the masking and keep just DINO alignment and expect the gains to survive.

(1) Add the modules one at a time; each one lowers gFID. Starting from a KL-VAE, the table adds random masking, then local alignment, then semantic masking, then global alignment (ImageNet 256, SiT-B, with decoder fine-tuning, optimal CFG).

gFID falls from 6.01 to 3.15. Random masking goes first and fixes the collapse, so the latent gets used at all — random masking alone gets to gFID 6.01. Adding local alignment then gives the single biggest further drop, to 3.53, by structuring the latent. Semantic masking adds discriminative content (to 3.32), and global alignment closes the gap (to 3.15). Four pieces, each pulling its weight — not DINO alone.

(2) How aggressive should the mask be? Sweeping the maximum mask ratio $M$ from 0.4 to 0.8, gFID (without CFG) falls then rises, and $M=0.7$ is best (14.59). Here "dino 100%" means every masked sample uses semantic masking, and "dino 50%" means half the samples use semantic masking and half use random masking. The 50/50 split (gFID 13.95) beats all-semantic (14.84): too weak a mask cannot force information in, too strong or too narrow a mask hurts reconstruction, and the two masks are complementary. The mild fidelity loss from strong masking is recovered by a separate decoder fine-tuning stage — encoder frozen, decoder trained 10 epochs without masks — which restores rFID to 0.43 and gFID to 13.90.

Q: do the ablations agree with the mechanism?

A: Yes. Add the modules in order and gFID drops 6.01 → 3.15 — random masking first lifts the latent out of collapse, then alignment and semantic masking improve it, exactly the order Section 5 predicts. The mask sweep agreeing — $M=0.7$ best, the even random/semantic split best — is the second confirmation.

Takeaway: The lever is not whether to use KL. It is making reconstruction need the latent (random plus semantic masking at $M\approx 0.7$, split evenly) and then structuring it with alignment, a result the latent visualization (Figure 3) and the probe (Figure 5) corroborate from the structure and discriminative sides.

BoundaryWhat the paper doesHow to read it

Q: what is this paper most likely to be oversold as?

A: As "masking solves collapse in general." The precise claim is narrower: in strong-compression continuous tokenization, masked reconstruction closes off the latent-optional solution, and DINO alignment injects the semantics, so 64 or 128 tokens reach high-fidelity generation.

Takeaway: The strength is strong compression with no collapse and high fidelity; the boundaries are DINO dependence, mask tuning, and the limited evaluation scope.

Closing: a few judgments

In one sentence: MacTok moves the fight against posterior collapse from tuning KL to changing the reconstruction task — masking makes reconstruction need the latent, and DINO alignment pours semantics into the few tokens.

Three takeaways:
  1. Because the cause is easy reconstruction, any future autoencoder can be stress-tested for collapse by ablating the latent and watching whether reconstruction stays flat.
  2. Making reconstruction harder, through random plus semantic masking, makes distortion depend on the latent, closes off the free move, and the collapse disappears.
  3. Global and local representation alignment pour DINO's discriminative semantics into the compressed latent, so the few tokens are both forced to be useful and rich in meaning.
