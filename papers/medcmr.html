Med-CMR 深读：把"医学多模态推理"这一个分数拆成 7 个维度，到底想逼出 MLLM 的什么短板？

Medical AI医学多模态 benchmarkMLLM 评测细粒度能力拆解LLM-as-Judge20,653 VQAMathJax checked

导读。当一个医学多模态大模型（MLLM）在某个评测上拿到 50 分，这个数字几乎什么都没告诉你——它是"看不清小病灶"丢的分，还是"看清了但推不出结论"丢的分？是常见病做不好，还是罕见病做不好？现有医学 VQA benchmark 多半停在"描述图像 / 检索一个显而易见的事实"，把这些异质的失败原因压成一个标量，于是模型的真实短板被平均掉了。Med-CMR 的核心赌注是"先拆维度，再评分"：把"医学复杂推理"显式拆成 3 个视觉维度（小目标检测、细节判别、空间理解）+ 4 个推理维度（时序预测、因果推理、长尾泛化、多源整合），每个维度配一类专门设计、强视觉依赖的题，让"哪种能力差"变得可定位。立题问题因此很具体：当你想知道一个 MLLM"离专家级临床推理还差多远"，到底该把这道题拆成几个可独立诊断的能力轴、又怎么保证拆出来的每一轴都真的吃图而不是吃文字？在 20,653 道经过"人审 + 模型筛"两阶段把关的题上评了 18 个 SOTA MLLM，结论是 GPT-5 居首（MCQ 57.81 / 开放式 48.70），但专科医学微调模型并不稳定优于通用强模型，而长尾泛化是所有模型共同的主导失败模式。

一句话贡献Med-CMR：首个把临床复杂推理拆成 7 个细粒度维度的医学多模态 benchmark；统一的 MCQ + 开放式双协议，开放式由外部 LLM 按四准则打分；20,653 道 VQA、覆盖 11 个器官系统 / 12 种成像模态，两阶段（人审 + 模型）质控。
关键机制不把"医学推理"当一个分数：视觉侧（小目标 / 细节 / 空间）× 推理侧（时序 / 因果 / 长尾 / 多源）= 7 条可独立诊断的能力轴；题目用模板保证"强视觉依赖 + 干扰项不与正解语义重叠"，并用三个 7B 模型"全做对就剔除"卡住难度下限。
核心证据GPT-5 七维全面第一、MCQ 57.81（图 1 雷达 / 表 2）；长尾泛化全场最难（最高仅 55.19，开源全部 < 46，表 2）；MCQ 准确率与模型规模强相关（$r$≈0.77–0.85），但开放式的视觉 grounding 与 GT 正确性对规模最不敏感（$r$=0.59 / 0.70，图 3）。
复现门槛纯评测、无训练账本：18 个模型经官方 API / VLLM 推理，开放式打分用 DeepSeek-V3.2-Exp 当 judge（与人审 Spearman $\rho$>0.78，最大 win-ratio 差 0.0449）。数据与 prompt 模板见项目页 LsmnBmnc/Med-CMR。

1. 核心问题：一个标量分数，藏起了所有真正难的临床场景

MLLM 正在从 demo 走向临床工作流。但论文开篇就点出一个评测层面的尴尬：要信任它们，光知道"答对的频率"不够，还得知道它们怎么得出结论——能不能发现细微病灶、整合多张图、追踪疾病演变、对罕见但要命的情形做推理？而绝大多数现有医学多模态 benchmark 只回答了这个问题的一小角：它们被感知层 VQA 主导——让模型描述一张图、或从一段短上下文里检索一个显而易见的事实。

  • 难的临床情形被"描述题"挡在门外。真正塑造临床决策的，是那些 setup 里藏起来的硬骨头：微小、低对比的病灶；跨模态比较；时序变化；连接"症状—影像—结局"的因果链；以及教科书里长尾分布的罕见病。感知层 VQA 看不到这些。
  • 一个分数 = 把异质的失败原因平均掉。把"医学多模态推理"当成单一标量去打分，等于默认所有错都是同一种错。可"看不清"和"看清了但推错"是两类完全不同的短板——前者要更强的视觉编码器，后者要更强的临床知识与多步推理。混在一个数字里，你既不知道病根、也无法对症。
  • 覆盖面与"是否吃图"没保证。很多题其实光读文字就能蒙对（视觉依赖弱），或只覆盖少数器官/模态。这样的分数对"真实工作流里的复杂推理能力"几乎没有可见度。

论文据此给出"做对这件事需要三样东西"：(1) 系统的能力拆解——不把"医学多模态推理"当一个分数，而是先把"细粒度视觉理解"与"下游推理"分开，再各自细分成临床上有意义的子维度；(2) 临床对齐且刻意做难的任务——题目围绕真实病例，显式瞄准时序、因果、长尾、多源这些难设定；(3) 广而精挑的覆盖——跨器官、模态、病程，且经专家审核保证题目真实可解释。

问：既然要评"复杂推理"，直接出一批难题、报一个总准确率不就行了？为什么非要先把能力拆成 7 个维度？

引导：盯住"总准确率"这个标量丢掉了什么——它把"为什么错"压没了。两个模型同样考 50 分，可能一个全栽在小目标、一个全栽在长尾，临床含义天差地别，但分数一样。

答：拆维度的本质，是把一个不可诊断的标量换成一组可独立归因的能力轴。这正是"先讲为什么需要、再讲是什么"的评测版本：先确认"我们到底想测什么能力、它会以哪几种方式失败"，再为每种失败造一类专门的题。一旦视觉侧（小目标 / 细节 / 空间）和推理侧（时序 / 因果 / 长尾 / 多源）被分开，"看不清"和"推不出"就不再互相掩盖——图 3 才能进一步揭示"MCQ 准确率随规模涨，但视觉 grounding 对规模几乎不涨"，图 4 才能把"医学微调到底改善了什么、又破坏了什么"讲清楚。拆维度不是为了好看，是为了让后续每一个诊断性结论有处落脚。

Takeaway：真问题不是"出多难的题"，而是"把分数拆成几条能独立归因的轴"。Med-CMR 的答案是 3 视觉 × 4 推理，配上"题必须吃图"的硬约束——拆得开、归得了因，才谈得上诊断模型短板。

2. 贡献拆解：作者明确提出了什么？

作者明确提出的贡献（论文 §1 三条）：

  • 首个支持临床复杂推理细粒度评测的 benchmark。Med-CMR 把推理拆成 7 个任务，每个捕捉一种不同的医学复杂度来源，提供结构化、临床落地的能力评估。
  • 统一的双协议评测。MCQ 测事实正确性，开放式测推理与生成质量；后者由一个外部、标准对齐的 LLM 沿 一致性 / 连贯性 / 视觉准确性 / GT 正确性 四准则打分，给出"答案对不对 + 推理好不好"的整体视图。
  • 18 个模型的横评 + 误差/消融分析。含医学专科 MLLM 与通用 MLLM；并实证给出"医学微调模型不稳定优于通用强模型"的具体失败模式与方向。

读数提醒：这篇是 benchmark 论文，不是方法论文

Med-CMR 不提出新模型、不训练任何网络，它的"方法"是benchmark 的构造与评测协议本身——维度定义、出题/造干扰项的流水线、双协议打分、judge 的可信度论证。所以本页的 #method 讲的是"怎么把临床复杂度拆成可评测的题"，#training 对应的是"评测设置而非训练账本"。把它当成一把诊断尺来读，而不是一个 SOTA。

读者能进一步推断的是：这篇真正推进的单一变量是"评测的分辨率"——它没换数据源（仍是病例报告 + 期刊文章）、没发明新打分函数（加权求和 + LLM-judge 都是现成思路），而是把"医学多模态推理"这一个分数，沿临床上有意义的 7 条轴切开，并诚实地把"长尾最难、医学微调有代价"作为结论摆出来。其价值不在"造了多难的题"，而在"让模型的短板第一次变得可定位"。

3. 数据：20,653 道题、11 器官系统 × 12 模态，源自真实病例报告

Med-CMR 的题目不是合成图、不是检索题，而是从公认生物医学期刊（如 JMCR、NEJM）的真实病例报告与研究文章里采集医学图像，连同人工标注的 caption 和元数据一起拿来。基于这些数据构造 7 类题，覆盖 11 个器官系统、12 种成像模态（影像之外还扩展到病理、眼科、内镜等）。器官系统分类遵循 Liachovitzky 的标准，模态分类遵循 DICOM 与 RSNA RadLex Playbook。

3.1 题量构成：16,655 MCQ（五选一）+ 3,998 开放式

20,653 道题里，16,655 道是五选一 MCQ、3,998 道是开放式。所有题归到 7 个主类别，每个主类别下再分 5 个子类型做更细的题型分类。MCQ 测"选对没"，开放式测"推理过程 + 最终结论"——两者合起来才覆盖"事实正确性"和"推理质量"两个面。

Med-CMR 的整条 curation pipeline 围绕一个目标：让每道题都难、都吃图、都临床真实。它分五段，每段都在堵一个特定漏洞。

④ 数据过滤人审：出题前医生过一遍所有图。模型筛：出题后用 Lingshu-7B / Qwen2.5-VL-7B / LLaVA-Med-7B 三个模型做，三个全做对的题剔除。三模型全对 = 太简单 → 剔除，给 MLLM 卡住难度下限。人审堵图质量，模型筛堵"送分题"。
⑤ 质量保证全科医生参与、专盯 LLM 生成内容，最终剔除 8% 可疑合成内容；人审阶段两名标注者联审 + 独立审计一致性，未达共识者删；四名标注者确认每题唯一无歧义正解；最后持照医师做医学正确性总审。多重把关堵"合成噪声 / 歧义题 / 多正解 / 医学错误"——benchmark 的可信度全靠这一关兜底。

问：用三个 7B 模型"全做对就剔除"来卡难度，这听起来像随手定的规则——它到底在控制什么变量？又有什么风险？

答：它控制的是"难度下限"这个混淆变量。如果题集里混了一堆送分题，模型间的差异会被"大家都对"稀释，benchmark 的分辨力下降——你想测的是"复杂推理"，结果一半分来自"认得出这是 CT"。用三个独立 7B 模型当筛子、"全对才剔"，等于给每道题设了一道"至少有一个弱模型答不出"的难度闸，把平凡题挡在外面。

但这一步也带进一个需要显式标出的风险假设：用模型当难度裁判，可能系统性偏向"对这三个模型难、但对人类未必难"的题，或反过来漏掉"三模型恰好都蒙对、其实很难"的题。论文用"只剔三个都对的"（而非"任一对就剔"）把这个偏置压到最小——保留了"至少一个模型不会"的题，难度闸是宽松下限而非严格筛选。这是"每个假设显式标出"的典型：难度由模型定，就得交代这个裁判可能引入什么偏。

Takeaway："三模型全对则剔"不是凑数规则，是给 benchmark 设难度下限、提分辨力的控制手段；代价是难度裁判由模型担任，可能轻微偏向"模型觉得难"的题——论文用"全对才剔"这个宽松阈值把偏置压到最小。

5. 评测协议：把"临床复杂度"拆成 7 维，再用四准则加权打分

5.1 七个维度：视觉 × 推理的二维分解

论文先论证一个前提：临床里感知与推理紧密耦合——医生在不确定证据、信息不全、需跨时间/上下文整合多模态证据的约束下做诊断。要系统地捕捉这些挑战，就把医学复杂度沿"视觉"和"推理"两个来源拆成 7 维：

  • 视觉侧（感知复杂度，源于图像的模糊与可变）：(1) 小目标检测——识别难以察觉的小/淡目标；(2) 细节判别——区分视觉相似但临床含义不同的发现；(3) 空间理解——整合并对齐多模态信息以保持空间一致。
  • 推理侧（高阶推断与不确定下的决策）：(4) 时序预测——对疾病进展/预后随时间推理；(5) 因果推理——用多步因果链连接症状、发现、结局；(6) 长尾泛化——罕见病样本极少时的决策；(7) 多源整合——从一个复杂病例里多个并存异常中抽取关键诊断线索。

这 7 维"反映医学复杂度的双重本质（感知 + 推理）"，并作为数据选择、出题、评测的概念骨架——前文流水线的每一步都挂在这 7 维上。

5.2 双协议：MCQ 测"选对没"，开放式测"推得好不好"

评测同时用两种题。MCQ 直接按正确率算。开放式则用一个基于准则的打分方案，同时评推理过程与最终结论——每个开放式回答沿四个预定义维度打分，最终分 $S$ 是四维的加权和：

$$S=\frac{\sum_{i\in\{\text{cons},\text{coh},\text{vis},\text{gt}\}} w_i\,s_i}{\sum_{i\in\{\text{cons},\text{coh},\text{vis},\text{gt}\}} w_i},\qquad w_{\text{cons}}{=}1,\ w_{\text{coh}}{=}1,\ w_{\text{vis}}{=}4,\ w_{\text{gt}}{=}4$$
式：开放式总分 $S$ 为四维度分 $s_i$ 的加权平均；权重把"视觉准确性 vis"与"GT 正确性 gt"各设为 4，"一致性 cons / 连贯性 coh"各设为 1。

四个维度各管一面：一致性（表述清晰、内部自洽）、连贯性（推理步骤间的因果衔接）、视觉准确性（识别/描述图像视觉特征的准确度）、GT 正确性（最终答案与 ground truth 的对齐）。所有开放式分数归一到 0–100 便于比较。

问：为什么把 vis 和 gt 的权重各设成 4、cons 和 coh 各设成 1？这 4:1 不是拍脑袋吗？给两条独立理由。

路径 A（权重对齐"真正难的能力"，自上而下）：§1 的整个立题，是要测"看得准 + 推得对"，而不是"说得漂亮"。一致性和连贯性衡量的是语言质量——回答读起来顺不顺、自不自洽；视觉准确性和 GT 正确性衡量的是临床实质——有没有看准图、最终答案对不对。如果四维等权，一个"满嘴流利但看错图、答错病"的回答会被语言分托起来，掩盖真正的失败。把 vis/gt 加权到 4，等于让总分主要由"实质"主导、语言只做次要调节，这正对应"先讲为什么需要"——权重跟着"我们到底想奖励什么能力"走。

路径 B（用数据自证这个权重不是空设，自下而上）：图 3 下行给了独立佐证：连贯性 $r$=0.79（随规模涨），而视觉准确性 $r$=0.59、GT 正确性 $r$=0.70（对规模最不敏感）。也就是说，cons/coh 这类语言维度已经相对成熟、容易随规模饱和，区分力弱；vis/gt 这类视觉-实质维度才是当前真正拉开差距、且堆参数也补不上的瓶颈。把高权重压在区分力强、最难的维度上，benchmark 的分辨力最高。两条路径殊途同归：一条说"我们想奖励实质而非辞藻"，一条说"实质维度本来就更难、更能区分模型"——4:1 因此不是拍脑袋，而是把分数的重心放到"难且重要"的能力上。

Takeaway：4:1 的权重把开放式总分锚到"看得准 + 答得实"，而非"说得顺"。图 3 进一步印证：语言维度易随规模饱和、区分力弱，视觉-实质维度才是真瓶颈——高权重压在真瓶颈上，分辨力最高。

5.3 LLM-as-Judge：用 DeepSeek-V3.2-Exp 当独立裁判，并自证可信

开放式由谁打分？论文用 DeepSeek-V3.2-Exp 作为独立评估者，在受控协议下打分，刻意避开被测模型自己当裁判（防"自我偏好"偏置）。这一步是 benchmark 可信度的命门：如果 judge 不可靠，所有开放式分数都站不住。论文因此专门做了一轮人-AI 对齐验证（见 §8）。

为什么"judge ≠ 被测模型"是机制必需，而非礼貌

LLM-as-Judge 的已知风险是自我偏好（self-preference）：模型倾向给"风格像自己"的回答打高分。若让被测的某个模型同时当裁判，它对自己/同族回答的打分会被系统性抬高，注入与"质量"无关的偏置。用一个不在被测列表里的独立模型（DeepSeek-V3.2-Exp）当裁判，等于把"裁判偏好"这个混淆变量从被测维度里剥出去。这与 §4 造干扰项时"换多个不同来源模型生成候选"是同一思路——用模型来源的多样性/独立性，抵消单一模型的系统偏置。

6. 评测设置（无训练账本）：18 个模型、官方 API / VLLM 推理

Med-CMR 不训练任何模型，所以这一节是评测设置而非训练账本。共评 18 个 MLLM：

  • 闭源（2）：GPT-5、Gemini-2.5-Pro。
  • 开源（16）：Gemma3-4B/27B、Medgemma-4B/27B、Qwen2.5-VL-7B/32B/72B、InternVL3.5-8B/38B/241B-A28B、Lingshu-7B/32B、Qwen3-VL-8B/32B/30B-A3B/235B-A22B。其中 Medgemma 与 Lingshu 系是医学专科微调模型，其余为通用 MLLM。

推理实现：开源模型用 VLLM 部署，闭源模型走官方 API。论文设计 prompt 模板约束各模型的输出格式，再用正则匹配抽答案；完整 prompt 模板在补充材料。开放式打分的 judge 是 DeepSeek-V3.2-Exp。

说明：本文是纯评测工作，不涉及 GPU 卡数 / 墙钟时间 / 学习率等训练超参——这些在原文中本就不存在，故此处不"论文未说明"地强行留空，而是指明工作性质决定了无训练账本。

8. 误差分析与消融：长尾为何主导失败？医学微调到底改了什么？

8.1 GPT-5 的三类失败（§4.1）

论文对上界模型 GPT-5 做细致误差分析：每个维度抽 100 个错例、手工归为五类（识别 / 推理 / 医学知识 / 题意误解 / 格式）。主导错因是图像识别、推理、医学知识不足三类，题意误解与格式问题罕见。三条观察：

  • 观察 1（识别错主导视觉密集维度）：GPT-5 常关注整体外观、忽略推理必需的细微细节，导致漏掉小特征、混淆重叠结构、跨切片追踪不一致。多见于小目标、多源整合、细节判别、空间理解。论文据此推断：当前视觉编码器缺乏鲁棒的多尺度特征与跨帧一致性，模型于是依赖粗糙的全局线索而非细粒度局部证据。
  • 观察 2（推理错源于"接不上证据"）：多数推理失败来自模型无法跨视图、跨时间、跨临床上下文连接证据，于是只盯部分信息、形成有偏的推理路径——即便部分观察是对的，结论仍错。最严重于空间理解、多源整合、时序预测、因果推理。
  • 观察 3（知识错卡在专科与罕见）：最先进模型仍栽在需要专科医学知识的题上，最明显于时序预测、因果推理、长尾泛化——正解依赖对疾病机制与罕见病的理解，而这些是模型训练时极少见的。

问：为什么"长尾泛化"会成为所有模型共同的主导失败模式，而不是某个模型的偶然短板？给两条独立理由。

路径 A（数据分布的必然，自下而上）：长尾的定义就是"罕见病样本极少"。无论预训练语料多大，罕见病在其中的占比天然极低——这是分布问题，不是规模问题。模型学到的是高频模式，遇到长尾就只能外推，而医学罕见病恰恰不能靠"长得像常见病"来蒙（图 1 长尾样题：CD117 染色模式里只有"弥漫强核染色提示异常定位"才指向罕见病，其余选项都是常见的合理干扰）。所以长尾难，难在训练分布里本就没有足够信号。

路径 B（与规模相关性的实证，自上而下）：图 3 显示长尾 MCQ 的 $r$=0.83，看似随规模涨；但表 2 的绝对值给出另一面：即便最大的 GPT-5，长尾也只有 55.19，开源全 < 46——规模能把长尾抬一点，却抬不过"勉强及格"。这与观察 3"知识错卡在长尾/罕见"一致：罕见病要的是专科知识的覆盖，而堆参数主要增强的是高频模式的拟合，对长尾的边际收益递减。两条路径殊途同归：一条说"训练分布里长尾信号本就稀薄"，一条说"实测里规模对长尾的提升撑不到及格线"——所以长尾是结构性的、跨模型的瓶颈。

Takeaway：长尾主导失败不是偶然，是"罕见病在任何训练分布里都稀薄"叠加"规模对长尾边际收益递减"的必然——这正是 Med-CMR 把长尾单列一维的价值：它逼出了一个堆参数解决不了的问题。

8.2 医学微调：MCQ 退化、开放式略胜（§4.2）

图 4(b) 给出一个跨所有医学微调模型的一致现象：相比各自的通用基线，它们在 MCQ 上退化（Sign/Wilcoxon $p$<0.001，差异显著），而在开放式上差距收窄、有时甚至反超（Sign $p$=0.45，差异不显著）。论文做了定向误差分析（采 150 个"Qwen2.5-VL-32B 对、Lingshu-32B 错"的 MCQ、150 个"Gemma-27B 对、Medgemma-27B 错"的 MCQ，及 50 个"Lingshu-32B 开放式胜 Qwen2.5-VL-32B"的题），找到两类 MCQ 错因：

  • (1) 方向对、细节漏：医学模型常走对了诊断大方向，却忽略关键的细微视觉线索，最终选错。
  • (2) 模式匹配过度：把少数显著特征直接连到典型诊断，不整合其他上下文信息。

两者合起来指向同一退化：医学微调让模型更依赖领域内的关联模式、更少做跨模态证据整合——通用多模态推理能力被磨损了。但开放式里，医学模型常产出医学语义更丰富的回答，即便漏了细粒度细节，也更可能生成"匹配整体视觉语义或合理诊断描述"的文本，于是在不那么吃精确性的开放式上略占便宜（Medgemma-27B 是例外，开放式全维下滑）。论文还把 500 道原本偏向通用模型的 MCQ 改写成开放式重评：Lingshu-32B 的开放式分反超其基线，而 Medgemma-27B 相对 Gemma-27B 仍下滑——印证"医学微调换来医学语义、却折损通用推理"这一权衡。

这个消融为什么能"归因"，而不只是"观察到现象"

关键在它的受控对照：每对都是"同源基线 vs 其医学微调版"（Gemma↔Medgemma、Qwen2.5-VL↔Lingshu），只改"是否医学微调"这一个变量，其余架构/规模尽量配平。再用"采基线对、微调错的题"做定向分析，把"退化具体退在哪"指出来。这正是"每次只改一个变量 + 主指标（MCQ 退化）+ 辅助指标（开放式收窄）双向验证"的设计——所以它能说"是医学微调导致了 MCQ 退化"，而非泛泛地"医学模型不如通用模型"。

8.3 judge 可信度：与人审强对齐（§4.3）

开放式既然靠 LLM-as-Judge，论文专门验证它可信。随机取 200 个开放式样本，收三个代表模型（Qwen3-8B、Qwen3-32B、MedGemma-27B）的回答，两名标注者按与 judge相同的准则独立排名、协商至一致；对每个模型对算 win-ratio（胜/平/负=1/0.5/0），再用 Spearman $\rho$ 量化人-LLM 排名一致性。结果（图 4(d)）：各维度 win-ratio 的最大差异仅 0.0449（出现在 MedGemma-27B 的连贯性维度）；一致性与视觉准确性的 $\rho$>0.8，连贯性与 GT 正确性的 $\rho$>0.78。论文据此判定：自动评测框架可作为专家评分的可靠替代。

问：用一个 LLM 给另一些 LLM 打分，凭什么不是"AI 自说自话"？这套验证哪一步真正堵住了循环论证？

答：堵点有三层。第一，裁判独立——DeepSeek-V3.2-Exp 不在被测 18 个模型里，从源头切断"自我偏好"。第二，人审用同一把尺——人类标注者按与 judge 完全相同的四准则排名，于是"人和 AI 是否一致"问的是"同一标准下结论是否一致"，而非两套互不相干的偏好碰巧重合。第三，双指标交叉——既看 win-ratio 的绝对差（≤0.0449，量"打分高低是否吻合"），又看 Spearman $\rho$（>0.78，量"排序是否吻合"）；一个查"分数对齐"、一个查"排序对齐"，两者都高才算可信。这三层合起来，把"AI 自说自话"的两个漏洞——裁判有私心、标准不可比——都显式堵上了。

Takeaway：LLM-as-Judge 的可信度不靠"信它"，靠"裁判独立 + 人审同尺 + win-ratio/Spearman 双指标交叉"三层验证把循环论证堵死——这是把"judge 可靠"当假设、再去证它的标准做法。

"医学微调有代价"的范围结论基于 Medgemma / Lingshu 两系。不宜外推成"所有医学微调都退化"——它说明的是这两类微调配方在复杂推理上有通用能力损失，换微调策略未必如此。
复现门槛纯评测，无训练；模型经官方 API / VLLM 推理，prompt 模板与数据见项目页。门槛低（不需训练算力），但闭源模型经 API评测——API 版本漂移可能让 GPT-5/Gemini 的数字随时间变化，复现需锁定模型版本与日期。

一句话概括复现门槛：这是一把"测疑难推理"的诊断尺，不是"测日常通量"的尺；它的可信度押在多重人审 + 单 judge 与人审的强对齐上，复现时要锁死闭源模型的 API 版本。

结语：几点学术判断

一句话总结：Med-CMR 把"医学多模态推理"这个不可诊断的标量，沿临床上有意义的 3 视觉 × 4 推理 = 7 条能力轴切开，配上"题必须吃图、干扰项不与正解语义重叠、三模型全对则剔"的构造流水线，再用 MCQ + 开放式（vis/gt 加权 4:1）双协议 + 独立 LLM-as-Judge（与人审 $\rho$>0.78） 把"答案对不对 + 推理好不好"分别量出来——于是在 18 个 SOTA 模型上，第一次能说清"GPT-5 强在哪、长尾为何全场最难、医学微调到底换走了什么"。

三个 takeaway

  1. "把分数拆成几条能独立归因的轴"才是 benchmark 的真正设计变量：拆成 7 维后，"看不清"（视觉侧）和"推不出"（推理侧）不再互相掩盖，图 3 才能揭示"规模买得到流利、买不到看准"，图 4 才能把医学微调的得失讲清楚。
  2. 权重要跟着"想奖励什么能力"走，并能用数据自证：开放式 vis/gt 设权 4、cons/coh 设权 1——上锚"看得准 + 答得实"，下有图 3 佐证（语言维度随规模饱和、区分力弱，视觉-实质维度才是真瓶颈）。4:1 不是拍脑袋。
  3. 长尾是堆参数解决不了的结构性瓶颈：罕见病在任何训练分布里都稀薄，规模对长尾边际收益递减（GPT-5 仅 55.19、开源全 < 46）。Med-CMR 把长尾单列一维，正是为了让这个问题无处可藏。

留给后续的两个反驳 / 验证路径：(a) "模型难度裁判 vs 人类难度"的直接对照——论文用"三模型全对则剔"卡了下限，但未验证"对这三个模型难"是否等于"对人类专家也难"；做一轮医生答题、与模型难度排名比 Spearman，能坐实难度闸是否对齐人类。(b) 把开放式 judge 从单模型扩成多 judge 投票，看 win-ratio 与 $\rho$ 是否进一步提升、单 judge 的盲区有多大——若多 judge 与人审对齐显著更好，则现有数字应被视为"可靠下界"而非定论。
