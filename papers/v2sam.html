V²-SAM: how to give SAM2 the missing target-view prompt for cross-view correspondence

Paper2Html2026-06-12CVPR 2026 Highlight · Top Paper candidateSource: arXiv:2511.20886v2 PDF
Cross-View Object CorrespondenceEgo-ExoSAM2 / DINOv3Multi-Prompt ExpertsPCCSMathJax checked

The short version. Cross-view object correspondence means taking an object marked in one camera view and segmenting the same physical object in another view. In the ego-exo setting, the query view may be egocentric, meaning first-person, while the target view may be exocentric, meaning third-person. SAM2, the Segment Anything Model 2, is strong at segmentation when it receives a prompt, where a prompt is conditioning information such as a point, box, mask, or visual reference. A coordinate prompt is a spatial point or box in the target image coordinate system; a visual prompt is an appearance embedding or reference cue. The difficulty is that cross-view correspondence needs the target-view coordinate, but that coordinate is exactly the unknown. V²-SAM synthesizes the missing target prompt with V²-Anchor, a training-free DINOv3-based geometric matcher, and V²-Visual, a learnable visual prompt matcher that transfers appearance through feature and structure paths. It then runs three experts, meaning three prompt-and-decoder routes, and uses Point-level Cyclic Consistency Selector (PCCS), a forward-backward point-consistency test, to choose the most reliable mask for each object. Intersection over Union (IoU) measures mask overlap, J&F averages DAVIS region similarity J and contour accuracy F, and the paper reports 48.0 total IoU on Ego-Exo4D, 78.8 J&F on DAVIS-17, and 77.2 IoU on HANDAL-X with 15.3M trainable parameters, about 1% of ObjectRelator's trainable-parameter count, while SAM2 and DINOv3 backbones stay frozen.

One-line contributionV²-SAM adapts SAM2 from single-view prompted segmentation to cross-view object correspondence by creating target-view coordinate and visual prompts from a query-view object mask.
Key mechanismV²-Anchor answers "where it is" with DINOv3 geometric matching and a sparse centroid prompt; V²-Visual answers "what it looks like" with VPMatcher, which aligns feature and structural cues across views.
Selection ruleAnchor, Visual, and Fusion experts each produce a target mask. PCCS sends each prediction back to the query view and picks the expert whose points are most cyclically consistent with the original query mask.
Main caveatThe 15.3M number is trainable parameters, not total model size. The multi-expert model has 543.4M total parameters, and its frozen SAM2-Hiera-Large and DINOv3 ViT-L/16 backbones are still required.

1. The core problem: SAM2 wants the answer as input

SAM2 works best when the prompt is spatially grounded. A target-view point tells the mask decoder where to look. A target-view box gives even more location and scale. In ordinary single-view segmentation, that is a reasonable interface because the user or upstream detector can provide a coordinate in the same image.

Cross-view correspondence breaks that interface. The query mask lives in one camera view, but SAM2 needs a prompt in the other camera view. Camera pose, viewpoint, occlusion, object deformation, and background clutter can all change at once. The target coordinate is not an auxiliary input anymore. It is the thing the task is trying to discover.

Q: why not use a visual reference and avoid coordinates entirely?

Guide: a visual reference can say what the object looks like, but it does not directly say where the object is in the target image.

A: That helps only when appearance remains discriminative across views. The paper argues that visual-reference prompting fails under large appearance change and in scenes with similar distractors. More importantly, dropping the coordinate prompt discards SAM2's strongest localization interface. V²-SAM instead asks whether a target-view coordinate can be synthesized from the query view, then combines that coordinate with visual cues rather than replacing one with the other.

Takeaway: The paper's central move is not to make SAM2 less prompt-dependent. It makes the missing prompt available.

2. What the authors actually claim

The paper's stated contributions are four. First, V²-SAM is presented as a unified framework that architecturally adapts SAM2 to cross-view object correspondence, especially ego-exo object correspondence. Second, it introduces two cross-view prompt generators: V²-Anchor for coordinate-based prompts and V²-Visual for appearance-guided prompts. Third, it builds a multi-prompt expert framework and PCCS to select the best expert per instance. Fourth, it validates the framework on Ego-Exo4D, DAVIS-2017, and HANDAL-X.

The largest reader-facing caveat is parameter accounting. The paper says the multi-expert model uses 543.4M total parameters and 15.3M trainable parameters. The "about 1%" comparison is valid only against ObjectRelator's 1587.3M trainable parameters, because V²-SAM freezes SAM2 and DINOv3 backbones.

3. Data and metrics: three ways to stress cross-view correspondence

The main task is query-driven target segmentation. The input is a query image or video frame $I_q$ with a query object mask $M_q$, plus a target image or video frame $I_t$. The desired output is the target mask $\hat M_t$ for the same object. Ego2Exo means the query is egocentric and the target is exocentric. Exo2Ego means the direction is reversed. The paper states that the model uses visual information only, without camera poses, semantic labels, or explicit 3D geometry.

Ego-Exo4D is the main ego-exo benchmark. HANDAL-X is a real-world manipulable-object dataset with 6-DoF pose annotations; the paper evaluates it in a zero-shot learning (ZSL) setting after training on Ego-Exo4D. DAVIS-17 is used as video object correspondence with a temporal gap of 20 frames, and the evaluation only considers objects visible in both frames.

The metrics are standard but easy to mix up. Mean Intersection over Union (mIoU or IoU) is mask overlap, higher is better. Continuity Accuracy (Cont.A) measures temporal or cross-frame continuity, higher is better. Localization Error (Loc.E) measures localization error, lower is better. DAVIS reports region similarity J and contour accuracy F; J&F is their mean, and higher is better.

4. Pipeline: two prompt generators, three experts, one selector

The design deliberately separates location from appearance. V²-Anchor is a geometry path: it tries to find a reliable target coordinate. V²-Visual is an appearance path: it tries to move region-level object identity across views. Fusion gets both prompts and lets the mask decoder reason with them jointly.

Q: why not average the three expert masks?

A: The paper uses hard selection rather than averaging because the experts fail in different ways. In cluttered scenes, a visual prompt can lock onto the wrong similar object. In dynamic scenes, a geometric prompt can lose the correspondence. PCCS treats each expert's output as a hypothesis and tests which one is most self-consistent across the two views.

Takeaway: The multi-expert design is useful only if the selector can identify the right expert per instance.

5. Method: making a coordinate prompt, then checking it cyclically

V²-Anchor: cross-view coordinate prompting

V²-Anchor uses DINOv3, a self-supervised vision feature extractor, because the paper treats its patch features as fine-grained and spatially aware. For each query patch, it computes cosine similarity against all target patches. The paper writes the dense heatmap as:

$$ \mathbf{H}_{ij}= \frac{\varphi(I_q)_i^\top \varphi(I_t)_j} {\lVert\varphi(I_q)_i\rVert_2\,\lVert\varphi(I_t)_j\rVert_2}, \qquad j^*(i)=\arg\max_j \mathbf{H}_{ij}. $$
Eq. 1: DINOv3 feature matching. $\varphi(I_q)_i$ and $\varphi(I_t)_j$ are query and target patch features; $j^*(i)$ is the best target patch for query patch $i$.

The query mask is projected onto the DINOv3 patch grid, so matching is constrained to foreground query patches rather than the whole image. The matched target points are then stratified so they do not all collapse into one local cluster:

$$ \mathcal{P}_q=\{\,i\mid (x_i,y_i)\in M_q\,\},\qquad \mathcal{P}_t=\{\,p_{j^*(i)}\mid i\in\mathcal{P}_q\,\}, $$ $$ \mathcal{P}'_t=\{\,p_i\in\mathcal{P}_t\mid \lVert p_i-p_j\rVert_2 \gt \tau,\ \forall j\lt i\,\}. $$
Eq. 2: foreground-constrained matches and sparse stratification. $\tau$ is the minimum pairwise distance; $\gt$ and $\lt$ are written explicitly for robust MathJax rendering.

The paper describes removing noisy correspondences and selecting a centroid point from the remaining matches. Written as a prompt-construction equation, that step is:

$$ \bar p_t=\operatorname{Centroid}(\mathcal{P}'_t) =\frac{1}{|\mathcal{P}'_t|}\sum_{p\in\mathcal{P}'_t}p,\qquad \tilde p_t=\Pi\!\left(\bar p_t;(H_t^{\mathrm{orig}},W_t^{\mathrm{orig}})\right), $$ $$ P_{\mathrm{anchor}}^{q2t}=\operatorname{PromptEnc}(\tilde p_t). $$
Eq. 3: centroid coordinate prompt. $\Pi$ maps the target-image coordinate into SAM2's canonical coordinate space, and $\operatorname{PromptEnc}$ is SAM2's prompt encoder.

This is the first major constraint to remember: V²-Anchor is training-free, but it is only as good as DINOv3's cross-view geometry. The paper's own expert analysis says Anchor Expert is strong in structured or relatively static scenes such as Cooking and Health, but weaker when appearance changes sharply, such as Music.

V²-Visual: appearance prompting with VPMatcher

V²-Visual handles the complementary cue. It starts by applying the frozen SAM2 image encoder $\phi(\cdot)$ and mask pooling to get region-level features:

$$ \mathbf{v}_q=\operatorname{MaskP}(\phi(I_q),M_q),\qquad \mathbf{v}_t=\operatorname{MaskP}(\phi(I_t),M_t). $$
Eq. 4: region-level query and target features. During training, $M_t$ is available as supervision.

The feature path builds a fused prompt embedding from the query feature and a mask encoding, then uses cross-attention to align it with image features. The structure path reconstructs a coarse target mask from a downsampled query mask and semantic modulation. The paper writes the structure modulation as:

$$ \tilde{\mathbf{m}} =\mathbf{m}_{\mathrm{prior}}\odot(1+\tanh(\gamma)) +\beta+F_{\mathrm{mask}}(M_q). $$
Eq. 5: structural mapping. $\mathbf{m}_{\mathrm{prior}}$ is the downsampled mask prior; $(\gamma,\beta)$ modulate it; $F_{\mathrm{mask}}(M_q)$ re-injects mask structure.

PCCS: point-level cyclic consistency

Point-level Cyclic Consistency Selector (PCCS) chooses among the three expert masks without using target labels. For each expert $k$, it treats the predicted target mask $\hat M_t^k$ as the new query and runs V²-Anchor back from target to query:

$$ \mathcal{P}_k^{t2q}= \operatorname{V^2\!-\!Anchor}(I_t,I_q;\hat M_t^k). $$
Eq. 6: backward point projection for expert $k$.

Then PCCS compares those back-projected points with reference points sampled from the original query mask. The paper states this as an average-distance proxy score. A compact selection criterion is:

$$ d_k=\frac{1}{|\mathcal{P}_k^{t2q}|} \sum_{p\in\mathcal{P}_k^{t2q}}\min_{r\in\mathcal{R}_q}\lVert p-r\rVert_2, \qquad k^*=\arg\min_{k\in\{\mathcal{A},\mathcal{V},\mathcal{F}\}}d_k, $$ $$ \hat M_t=\hat M_t^{k^*}. $$
Eq. 7: PCCS selection. $\mathcal{R}_q$ is the sampled reference-point set from $M_q$; $\mathcal{A}$, $\mathcal{V}$, and $\mathcal{F}$ denote Anchor, Visual, and Fusion experts.

Why point-level selection matters

Prior cyclic selection reconstructs a query-view mask and compares masks. PCCS avoids that extra decoding pass. In the paper's Table 10, the two-expert setting improves Exo2Ego IoU from 46.73 to 48.17 while reducing runtime from 620 ms/sample to 510 ms/sample. In the three-expert setting, it keeps accuracy essentially unchanged and still reduces runtime and FLOPs.

6. Training: what is frozen, what is learned

The base segmentation model is SAM2-Hiera-Large, and the geometric feature extractor is DINOv3 ViT-L/16. The paper says the SAM2 encoder is frozen. It also keeps the coordinate-only Anchor Expert training-free because coordinate prompts are already supported by the pretrained SAM2 prompt and mask decoder. The learnable parts are the VPMatcher and the mask decoders for the Visual and Fusion Experts. That is why the paper can report 15.3M trainable parameters for the multi-expert system while still using large frozen backbones.

$$ \mathcal{L} =\lambda_1\mathcal{L}_{v}(\hat{\mathbf{v}}_c,\mathbf{v}_t) +\lambda_2\mathcal{L}_{s}(\hat M_c,M_t) +\lambda_3\mathcal{L}_{m}(\hat M_t,M_t). $$
Eq. 8: training objective. $\mathcal{L}_v$ is visual contrastive loss, $\mathcal{L}_s$ is structural constraint loss, and $\mathcal{L}_m$ is mask prediction loss.

The default loss weights are $\lambda_1:\lambda_2:\lambda_3=1:1:10$. During the first 4K training steps, the paper sets $\lambda_1=100$ to strengthen contrastive feature representation. Mask prediction uses cross-entropy plus Dice loss; the supplementary model table states binary cross-entropy weight 2.0 and Dice weight 0.5.

9. Limitations and reading boundaries

Boundary checklist. Caveats stated or directly implied by the paper.
BoundaryWhat the paper saysHow to read it
Frozen backbonesV²-SAM uses SAM2-Hiera-Large and DINOv3 ViT-L/16, with the SAM2 encoder frozen.The 15.3M parameter claim is trainable-parameter accounting. It is not the whole model footprint.
DINOv3 dependenceV²-Anchor relies on DINOv3 geometry-aware patch features.If DINOv3 fails to match the object across views, the Anchor Expert can fail. The Visual and Fusion experts are the fallback, not a fix to DINOv3 matching.

Inference costAll three experts operate in parallel during inference, and PCCS selects among them at batch size 1.The best accuracy uses multiple decoders plus selection. Table 10 reports 760 ms/sample and 2184.63G FLOPs for A+B+C with Cycle-Points.
Code and modelsThe PDF states that codes and models will be released at V²-SAM.Release completeness, checkpoints, and data scripts are not stated in the paper text used for this page.

Q: what is the most likely overclaim?

A: Saying that V²-SAM is a 15M-parameter SAM2 replacement. The paper does not say that. It reports 15.3M trainable parameters because large backbones are frozen, and the multi-expert model still has 543.4M total parameters.

Takeaway: The right claim is parameter-efficient adaptation, not a tiny standalone segmentation model.

10. Summary: the useful idea is prompt recovery

V²-SAM is best read as a prompt-recovery paper. SAM2's coordinate prompt is powerful, but cross-view correspondence removes it. V²-Anchor reconstructs that missing coordinate from DINOv3 geometric matches and turns it into a centroid prompt. V²-Visual transfers appearance through VPMatcher. The three-expert design then accepts that no single prompt type wins everywhere, and PCCS supplies a lightweight per-instance selector.

The evidence is broad enough to support the paper's main claim: V²-SAM reaches state-of-the-art IoU or J&F on Ego-Exo4D, DAVIS-17, and HANDAL-X among the listed baselines. The honest boundary is just as important. It loses the Loc.E columns on Ego-Exo4D, depends on frozen SAM2 and DINOv3 backbones, and assumes enough cross-view visibility for correspondence and cyclic consistency to be meaningful.

Three takeaways

  1. When a foundation model fails after a task shift, identify the missing input before redesigning the model.
  2. For cross-view SAM2, the missing input is a target-view coordinate prompt, not only a better visual reference.
  3. Expert selection matters because geometry and appearance fail on different scene types.
